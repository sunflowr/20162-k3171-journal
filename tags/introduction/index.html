Module4 introduction

October 18, 2016 · 3 minute read · module4

  • Tags
  • introduction
  • lecture

Today we were presented with Module 4 - the last module for the course. The topic for the module is Body and Sound.

Keywords for the modules are:

  • Tightness - The user can see a direct corelation between its interaction and the feedback from the artifact.
  • Ambiguity - Some form of uncertainty in the inner workings, making the artifact feel more alive.
  • Openness - There’s no fixed state of the interaction, no start/stop condition. It’s always sensing the environment and reacting to it. Much like how we humans and most - if not all - living beings.

Material for the module consist of:

  • Ableton Live with Max for Live
  • Arduino using Firmata sketch
  • Max two analog sensors
  • A few pre-selected audio files.

Idea-generation

Instructions:

  1. Ideate -> Pick two (sets of) everyday movements.
  2. Explore the characteristics of these two (sets of) movements with your body.
  3. Choose one set of movements - work with it, identifying 3-5 of the movements characteristics.
  4. Make brief description of each of these characteristics.

Movements

Movements in bold was the once we expanded on.

  • Shaking hands
  • Opening doors
  • Drinking coffee
  • Smoking
  • Read/write
  • Flipping pages
  • Put on / taking off shoes
  • Pick up thing from the floor
  • Hugging (man-hug - the awkwardness)
  • Sitting in a office-chair
  • Arm-wrestling

Drinking coffee

  • Holding (the cup)
  • Grabbing
  • Lifting
  • Tilting
  • Rotating

We thought these were some interesting interactions that we could apply sensors to feel, but after talking to a few other groups it felt like everyone was going to do something similar. So we decided on our other options - sitting in a office chair.

Sitting in a office-chair

  • Spinning (the chair)
  • Leaning left/right/back/forward
  • Sitting down
  • Standing up

We thought this could be an fun thing were we can play around with how people sit in the chair, like how we all re-position our self from time to time without really thinking about it. We’re thinking that we could make the person aware with the way the they use the chair. By sensing how the person lean the chair or position it we could either create a feelings such as power or discomfort by playing with the sounds.

Tech

We decided we want to put a sensors for sensing the rotation of the chair as the person using it re-position them self and the chair. We think the most viable solution for this is to stick a potentiometer on the chair and connect it between the two rotating parts of the chair. To stop the potentiometer from breaking if the user would do a 360 turn we’ve removed the mechanical part in the potentiometer that stops it from rotating a full turn.

We also decided on using a accelerometer from TinkerKit as the one provided by the teacher communicate using I2C. The accelerometer provide sensors for the X, Y and Z axis. The idea is to be able to sense the bodily movement of the person using the chair.

Module3 introduction

October 3, 2016 · 2 minute read · module3

  • Tags
  • introduction

Today module 3 was presented to us. The keyword is ‘Nuance’ and as far as I interpreted it, it’s abut expressiveness and flow in the interaction. To simplify step based interactions into something much more fluid and natural feeling.

Material / requirements for the module is touch and pixels. Target device can be anything phone, tablet, billboard, printer, something that don’t exist yet.

Lecture

Skillful coping

  • To keep up with a situation.
  • Things seem to “fall into place”.
  • We do just the right thing at the right moment.
  • Seem to flow, don’t need to know about the how.

Ideas

  • Start with something with limited nuance.
  • Look for interactions that involve repeated steps, or selection from a menu menu option.
  • See if you can “collapse” it down into a single expressive action.

Material quality

  • Maybe pixels can be used and feed-forward in a abstract non-symbolic way?
  • How could touch be used more for continuous fluid interaction rather than poking fake buttons?

User stories

Example 1

MAH printer system requires multiple steps in order to print, even in the case where the user want’s to print everything, or the there’s only one thing in the queue. It would be desirable if the user could in a easy manner skip this friction in the interaction and create a flow, making the experience better and give the user more expressiveness in it’s way of commanding the printer.
