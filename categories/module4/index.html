Module4 presentation

October 27, 2016 · 2 minute read · module4

  • Tags
  • presentation

We had decided on meeting up a few minutes earlier to test things out and calibrate sensors and setup Ableton and Max for Live. I was a bit late because I had problem finding the DVI/HDMI adapter for the computer, but we still managed to do the things we planned.

I think the presentation went fine for most part. I agreed with the feedback that it would have been much better if we had said the text instead of showing it in the video (see Video).

It also seemed like we failed to convey that we tried lots of different combinations of how we used the sensors before ending up were we currently are.

We should have been more clear about important finding such as the breaking point were realized best practice (for us in this special case) for using the accelerometer.

As we didn’t get any real critic of the openness, tightness or ambiguity we assume that we applied it in a good way in the interaction.

Learnings

  • Through the sketching process we realized that the best way to use the accelerometer is to stop thinking of it as absolute values and instead use the shifting values over time to to either increase or decrease something.

  • Another important realisation is that “less is more” still holds true. After presenting the current sketch to the teachers a few days before the presentation - thinking that we had something that felt good - we quickly realized that the use of so many different sensors and sounds in combination created something very complex and hard to control. The only reason we thought it was “ok” was because we were so use to it and knew how it worked. After that we tried simplifying the design but removing sounds and simplifying the usage of the accelerometer.

  • It’s important to setup and explain the limitations (rules) to the person interacting with the artifact - if there’s something that could break it. As an example if the person using the chair didn’t sit on it with the Y-axis of the accelerometer facing forward when sitting down, the readings would be all wrong.

Sketching

October 21, 2016 · 4 minute read · module4

  • Tags
  • status

Through most of the day we experimented with applying different effects (volume, panning, filter, delay, reverb, …) to the sounds and changing different parameters using the X and Y accelerometer sensors and the potentiometer. We discovered that it was hard to noticed if the user was changing the sounds or if the sounds where changing them self. The sounds included very dynamic elements and were already very highly processed with reverb, delays and filters that it was hard to apply anything like this and change the sound subtly.

It would have been much better if we were allowed to use raw unprocessed waveforms or the sound devices included in Ableton. It would have allowed for more changes. Take a square-wave, we could have mapped sensors to pulse-width and pitch and create lots of changes in the sound without even using filters

Sketching Z-axis

I had previously wondered about the third axis - the Z axis - as there were pin-header for connecting it to the Arduino. Today we discovered that actually there were solder holes for soldering in wires to 5V, GND, X,Y and Z. Apparently we had to solder a wire to access the Z-reading.

It was kind of funny because the Z-axis actually behaved more like we expected a accelerometer to behave. Always striving for a value and when affected with movement jump in the values but as soon as it was still go back towards this middle-value.

After fixing this we started trying out the Z-axis applying a frequency-shifting effect when force were applied to Z. This turned out to work so satisfyingly well that we held on to that functionality for the rest of the day. The feedback felt so “natural” and expected in the way the sound went up and down in pitch with the up and down movement of the chair.

We felt a certain tightness to the interaction and how it behaved, it always seemed to behave the same way and it was easy to understand. But thanks to gravity and precision in motor skills it’s hard to feel you have a exact control of it. One of the teaches suggested it made it more ambiguous than tight. As I see tightness as being the about the experienced tightness between the interaction and the feedback from the system and the user understanding the correlation. The more clear this is to the user, the tighter the interaction feels.

“The prevailing buttons in Snoezlen mostly react instantaneously; that is, cause and effect happen (almost) at the same time, but the buttons separate cause and effect with regards to location and amount of feedback. […]

Attributes of tightness play out differently across the designs. Both MalleablePillow and ActiveCurtain give feedback instantaneously where they are touched, and they react proportionally.”

(Larsen, 2015)

Ambiguousness is about a lack of understanding of how something work, causing the user to guess and in best of cases explore to get a better (deeper) understanding of the feedback from the interaction/artifact/system.

“By impelling people to interpret situations for themselves, it encourages them to start grappling conceptually with systems and their contexts, and thus to establish deeper and more personal relations with meanings offered by the system”

(Gaver, Beaver, & Benford, 2003)

I’d argue that the two don’t necessarily have to be in odd to each other in this case. The user can experience a direct result of sitting on the chair - tightness:

Playing of a sound and hearing it change in pitch with the movement.

But there’s still the question if the user instantly realize that moving up and down changes the pitch of the sound or if this is something that will be realized through exploration - ambiguity.

Sensing the chair

October 20, 2016 · 2 minute read · module4

  • Tags
  • status

Today we fixed with the chair. The idea was to use a typical office chair unfortunately it turns out it was hard to attach the potentiometer to them. After looking around for different solutions to this and talking to the people in the workshop area we found some chairs that we were able to pull apart easier to attach the potentiometer.

In order to fix the potentiometer to the chair we pulled it apart and took a wooden block and attached it to the chair and drilled a hole for the potentiometer to sit in (see attached photos).

We had a few ideas on how to the actual connection between the turning part of the chair and the potentiometer. But most of them required much work to get it working. In the end we realized we could just do it similar to how old belt-driven record players work. So we put a elastic around the rotating part and the potentiometer and rely on friction to turn the knob. It seemed to work really well so we went with this idea.

We quickly realized it was better to attach the accelerometer to the chair instead of the user. It fixed issues regarding how the user would have been limited in the movement by the wires attached to the accelerometer. In this way it also put the focus all on the chair and allowed for more flowing openness for the users. The user now just had to sit down without first having to attach some device before using the artifact.

After this we finally felt ready to start sketching.

Sketching

Most of todays sketches was about getting to know the sensors and what we can expect from them and how to use them.

We played around with X and Y axis mostly as we found them very confusing. We expected them to be continuously changing with the movement (or lack of movement). But instead it seemed to give us something that looked more like absolute values that you would expect from a gyroscope. We did lots of tests and felt that this behaviour seemed to happen each time so we decided on using it in this more “gyroscopic” way, using the values to measure how far the user were leaning in a direction.

Module4 introduction

October 18, 2016 · 3 minute read · module4

  • Tags
  • introduction
  • lecture

Today we were presented with Module 4 - the last module for the course. The topic for the module is Body and Sound.

Keywords for the modules are:

  • Tightness - The user can see a direct corelation between its interaction and the feedback from the artifact.
  • Ambiguity - Some form of uncertainty in the inner workings, making the artifact feel more alive.
  • Openness - There’s no fixed state of the interaction, no start/stop condition. It’s always sensing the environment and reacting to it. Much like how we humans and most - if not all - living beings.

Material for the module consist of:

  • Ableton Live with Max for Live
  • Arduino using Firmata sketch
  • Max two analog sensors
  • A few pre-selected audio files.

Idea-generation

Instructions:

  1. Ideate -> Pick two (sets of) everyday movements.
  2. Explore the characteristics of these two (sets of) movements with your body.
  3. Choose one set of movements - work with it, identifying 3-5 of the movements characteristics.
  4. Make brief description of each of these characteristics.

Movements

Movements in bold was the once we expanded on.

  • Shaking hands
  • Opening doors
  • Drinking coffee
  • Smoking
  • Read/write
  • Flipping pages
  • Put on / taking off shoes
  • Pick up thing from the floor
  • Hugging (man-hug - the awkwardness)
  • Sitting in a office-chair
  • Arm-wrestling

Drinking coffee

  • Holding (the cup)
  • Grabbing
  • Lifting
  • Tilting
  • Rotating

We thought these were some interesting interactions that we could apply sensors to feel, but after talking to a few other groups it felt like everyone was going to do something similar. So we decided on our other options - sitting in a office chair.

Sitting in a office-chair

  • Spinning (the chair)
  • Leaning left/right/back/forward
  • Sitting down
  • Standing up

We thought this could be an fun thing were we can play around with how people sit in the chair, like how we all re-position our self from time to time without really thinking about it. We’re thinking that we could make the person aware with the way the they use the chair. By sensing how the person lean the chair or position it we could either create a feelings such as power or discomfort by playing with the sounds.

Tech

We decided we want to put a sensors for sensing the rotation of the chair as the person using it re-position them self and the chair. We think the most viable solution for this is to stick a potentiometer on the chair and connect it between the two rotating parts of the chair. To stop the potentiometer from breaking if the user would do a 360 turn we’ve removed the mechanical part in the potentiometer that stops it from rotating a full turn.

We also decided on using a accelerometer from TinkerKit as the one provided by the teacher communicate using I2C. The accelerometer provide sensors for the X, Y and Z axis. The idea is to be able to sense the bodily movement of the person using the chair.
