Idea generation

October 5, 2016 · 3 minute read · module3

  • Tags

Ideas

Limitations

  • Platform - mobile phone
  • Use touch gesture
  • Use pixels as indicator

Force-touch - hold to call favorite

When user force touch over a certain threshold on the Phone icon the user is given the option to call a favorite contact without having to go through the contact book.

Multi-touch swipe - skip X pages

Depending on amount of fingers touching the screen when swiping skip X amount of pages. E.g. when flicking through the app pages, instead of swiping three times to go from page 1 to page 3, swipe with two fingers to jump forward two pages instead.

Swipe gesture on the “Home button”

Considering that the “home button” now day seem to be smarter than just a regular button (e.g. finger recognition) we wondered if it would be possible to some how detect gestures on it.

How could interacting using gestures on this button rather than the screen aid in everyday actions on the phone?

Interactive billboards

What would happen if public billboards were more like gigantic interactive screens?

How would it handle multiple users?

Would it make people take more notice to and engage more in the message on the billboard (e.g. even if it were a commercial for toothbrushes).

Pinching for overview

Similar to how it’s used on OSX give the user a overview screen of all the active applications; doing this on the phone could as an example present the available pages on the home screen of the phone for faster movement.

Misc ideas

  • Is there any reason for not interacting using the foot more?

Selected idea - Fast scroll / filter scroll.

After a bit more discussions and consulting with the teacher we ended up with a combination of some ideas that we evolved into our current idea. We aim to create something that could be used in context of social media, news site and other sites that are based on articles/posts.

The idea is to allow the user to do fast scroll / filter scroll to quickly skip articles in the article feed based on some search criteria.

In order to do this we split the screen into multiple areas. One area functions like normal and allow the user to scroll normally, the other area(s) act as zones that react on swipe up/down and react by searching for next/previous post using the selected search criteria.

The idea is to allow the user to for example skip posts in social media to jump to next post for a certain friend, page or group without having to scroll forever.

We also have some extra ideas with using force-touch to scroll within post comments that we’re considering trying.

Module3 introduction

October 3, 2016 · 2 minute read · module3

  • Tags
  • introduction

Today module 3 was presented to us. The keyword is ‘Nuance’ and as far as I interpreted it, it’s abut expressiveness and flow in the interaction. To simplify step based interactions into something much more fluid and natural feeling.

Material / requirements for the module is touch and pixels. Target device can be anything phone, tablet, billboard, printer, something that don’t exist yet.

Lecture

Skillful coping

  • To keep up with a situation.
  • Things seem to “fall into place”.
  • We do just the right thing at the right moment.
  • Seem to flow, don’t need to know about the how.

Ideas

  • Start with something with limited nuance.
  • Look for interactions that involve repeated steps, or selection from a menu menu option.
  • See if you can “collapse” it down into a single expressive action.

Material quality

  • Maybe pixels can be used and feed-forward in a abstract non-symbolic way?
  • How could touch be used more for continuous fluid interaction rather than poking fake buttons?

User stories

Example 1

MAH printer system requires multiple steps in order to print, even in the case where the user want’s to print everything, or the there’s only one thing in the queue. It would be desirable if the user could in a easy manner skip this friction in the interaction and create a flow, making the experience better and give the user more expressiveness in it’s way of commanding the printer.
