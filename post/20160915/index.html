What was done yesterday.

We added a new background again to see if this could take us in the direction we wanted to go with the experience. To some extent it felt closer to the initial idea.

After the rewriting of rendering to use DOM/CSS instead of JS canvas there were further enhancements made today so the object resize as the background is “zoomed”, thus creating a feeling of depth.

Issues

After trying with real data from the teachers firebase account it became clear that there might be some performance issues when rendering on a phone.

I’ve done some small optimization but nothing that gives that needed boost.. the best option so far is to put a cap on amount of found devices to render.
